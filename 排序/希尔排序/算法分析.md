## 基础模板的实现细节

<br />

### 1.最外层循环：控制“步长”

```cpp
for (int gap = n / 2; gap > 0; gap /= 2)
```

- **核心逻辑**：希尔排序的本质是“分组插排”。这一层决定了分组的**跨度**。

- **过程**：
    - 初始时 ``gap`` 很大（例如数组长度的一半），元素之间相距很远，进行粗粒度的排序。
    - 每次循环 ``gap`` 减半，分组变细，元素移动的距离变短。
    - 最后一次 ``gap = 1``，此时退化为标准的插入排序，但因为前面的几轮排序已经让数组“基本有序”，这最后一轮会非常快。

<br />

### 2.最巧妙的设计：中层循环

```cpp
for (int i = gap; i < n; i++)
```

这是最容易让人困惑的地方。**为什么 ``i`` 是从 ``gap`` 开始，而不是从 ``0`` 开始？且 ``i`` 是连续递增的？**

- **误解**：很多人以为希尔排序是先把第一组排完，再排第二组。如果那样写，代码需要嵌套两层循环（先遍历组，再遍历组内元素），很麻烦。

- **真相**：这段代码采用了“**轮流处理**”的策略。
    - 假设 ``gap = 3``，数组被分为 3 组：
        - 第 0 组：下标 ``0, 3, 6...``
        - 第 1 组：下标 ``1, 4, 7...``
        - 第 2 组：下标 ``2, 5, 8...``
    - 当 ``i = gap`` 时（即 ``i=3``），处理的是**第 0 组**的第二个元素。
    - 当 ``i = 4`` 时，处理的是**第 1 组**的第二个元素。
    - 当 ``i = 5`` 时，处理的是**第 2 组**的第二个元素。
    - 当 ``i = 6`` 时，又回到了**第 0 组**的第三个元素。

**结论**：通过让 ``i`` 从 ``gap`` 连续递增到 ``n-1``，代码巧妙地在一次循环中**交替地处理了所有分组**。这极大地简化了代码结构。

### 3.最内层循环：插入排序的变种

```cpp
int temp = arr[i];
int j = i;
while (j >= gap && arr[j - gap] > temp) {
    arr[j] = arr[j - gap]; // 后移
    j -= gap;              // 关键：向前跳跃 gap 步
}
arr[j] = temp;
```
这就是标准的插入排序逻辑，唯一的区别在于**步长**：

- **标准插入排序**：``j`` 每次减 ``1``（``j--``），因为它是和**相邻**的前一个元素比较。
- **希尔排序**：``j`` 每次减 ``gap``（``j -= gap``），因为它是和**同组**的前一个元素比较。
