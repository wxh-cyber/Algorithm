## 区间映射公式的推导

在进行桶排序时，对于单个的元素，为了确定其所在的桶，需要进行区间映射。

区间映射公式为：
```txt
index = (value - min) * (n - 1) / (max - min)
```

### 1. 算出“相对比例”（归一化到 0~1 之间）

假设最小值是 ``minVal``，最大值是 ``maxVal``，当前值是 ``val``。

- 当前值距离最小值有多远？ -> ``val - minVal``
- 整个数据的总跨度有多大？ -> ``maxVal - minVal``

两者相除，就得到了当前值在整体数据中的**比例位置**：

``(val - minVal) / (maxVal - minVal)``

这个结果一定在 ``0`` 到 ``1`` 之间。当 ``val`` 是最小值时为 0，当 ``val`` 是最大值时为 1。

### 2. 映射到桶的索引

假设我们有 ``bucketCount`` 个桶（比如 5 个桶），它们的索引分别是 ``0, 1, 2, 3, 4``。

我们只需要把上面算出的比例，乘以最大索引数（``bucketCount - 1``），就能得到对应的桶号：

``(val - minVal) / (maxVal - minVal) * (bucketCount - 1)``

### 3. 为什么是乘以 (bucketCount - 1) 而不是 bucketCount？

这是为了**防止数组越界**。

- 如果乘以 ``bucketCount``：当 ``val`` 刚好等于 ``maxVal`` 时，比例是 1，计算结果就是 ``1 * bucketCount = bucketCount``（比如 5）。但桶的索引最大只有 ``4``，访问索引 ``5`` 就会越界崩溃。
- 如果乘以 ``bucketCount - 1``：当 ``val`` 等于 ``maxVal`` 时，结果就是 ``1 * (bucketCount - 1) = 4``，刚好落在最后一个桶里，完美贴合边界。
