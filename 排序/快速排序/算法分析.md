# 算法分析

<hr />

## 快速排序基础版的``partition``

这个基础版本使用的划分方法叫做 **Lomuto 划分方案**。它的核心思想非常直观：**维护一个边界，把小于等于 pivot 的元素不断扔到边界左边，大于 pivot 的元素留在右边**。

<br />

### 1.划分的最终目标

给定一个数组区间 ``[low, high]``，我们选择最后一个元素 ``arr[high]`` 作为 **pivot（基准值）**。

执行完划分函数后，必须达到这样的效果：

1. **pivot** 被放到了它最终排好序时应该在的位置（设该位置为 ``k``）。
2. 位置 ``k`` 左边的所有元素都 ≤ **pivot**。
3. 位置 ``k`` 右边的所有元素都 > **pivot**。

注意：左边元素不需要有序，只要都小于等于 pivot 即可；右边同理。

<br />

### 2.两个指针的角色

Lomuto 方案使用了两个指针 ``i`` 和 ``j``，它们各有明确的分工：

- **``j``（探索者）**： 从 ``low`` 走到 ``high-1``，负责遍历数组中除 pivot 外的每一个元素，审视它们的大小。
- **``i``（边界维护者）**： 初始值为 ``low - 1``。它始终指向**已探明的小于等于 pivot 的区域的最后一个元素**。换句话说，``i`` 划定了一条边界：``i`` 及其左边的元素都是 ≤ pivot 的，``i`` 右边到 ``j`` 之间的元素都是 > pivot 的。

<br />

### 3.核心逻辑：不变量

在 ``j`` 从左向右遍历的过程中，内存中的数组状态始终维持着这样一个**不变量**：
```txt
[  <= pivot  |  > pivot  |  未探查  | pivot ]
 ^           ^           ^          ^
low         i+1         j          high
```
- ``[low, i]``：已探明的 ≤ pivot 的区域。
- ``[i+1, j-1]``：已探明的 > pivot 的区域。
- ``[j, high-1]``：尚未探查的区域。
- ``arr[high]``：pivot 本身。

算法的每一步操作，都是为了在 ``j`` 向右移动时，维持这个不变量。

<br />

### 4.具体操作步骤

当 ``j`` 遍历到元素 ``arr[j]`` 时，只有两种情况：

<br />

**情况 A：``arr[j] <= pivot``**

- **动作**： 我们需要把这个小元素放到左边的区域。因此，先把左区域的边界扩大一格（``i++``），然后把 ``arr[j]`` 和 ``arr[i]`` 交换。
- **效果**： 原来的 ``arr[i+1]``（本就是 > pivot 的元素）被换到了右边，而新的 ≤ pivot 的元素被加到了左边。不变量依然成立。

<br />

**情况 B：``arr[j] > pivot``**

- **动作**： 什么都不用做。大元素本来就应该呆在右区域，直接让 ``j++`` 继续探索即可。
- **效果**： 右区域自然扩大了一格。不变量依然成立。

<br />

**最后一步：归位 pivot**

- 当 ``j`` 走到 ``high`` 时，循环结束。此时除了 pivot，所有元素都已探明。
- 当前状态：``[low...i]`` 都 ≤ pivot，``[i+1...high-1]`` 都 > pivot。
- 显然，pivot 的正确位置应该是 ``i+1``。我们把 ``arr[i+1]`` 和 ``arr[high]``（也就是 pivot）交换。
- 这样，pivot 就到了最终位置，左边全小，右边全大。返回 ``i+1``。
