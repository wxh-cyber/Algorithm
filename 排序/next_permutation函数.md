# ``next_permutation``函数

<hr />

``next_permutation`` 是 C++ 标准库 ``<algorithm>`` 中的一个非常实用的函数。顾名思义，它的作用是求当前序列的“**下一个排列**”。

<br />

## 1.什么是字典序？

字典序就像我们在字典里查单词的顺序，从左到右逐位比较，小的排在前面。

对于数字序列 ``1, 2, 3``，它的所有全排列按字典序排列如下：

1. ``1 2 3`` （最小）
2. ``1 3 2``
3. ``2 1 3``
4. ``2 3 1``
5. ``3 1 2``
6. ``3 2 1`` （最大）

<br />

## 2.``next_permutation`` 的作用

当你把一个序列（比如数组或字符串）传给 ``next_permutation`` 时，它会**直接在原序列上修改**，把它变成按字典序的下一个排列。

**它的返回值非常重要**：

- 如果当前序列不是最大的（没有变成降序），它会成功生成下一个排列，并返回 ``true``。
- 如果当前序列已经是最大的排列（即完全降序，比如 ``3 2 1``），它会把序列重置为最小的排列（升序 ``1 2 3``），并返回 ``false``。

<br />

## 3.代码示例

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int a[3] = {1, 2, 3};
    
    // 打印当前初始排列
    cout << "初始: " << a[0] << " " << a[1] << " " << a[2] << endl;
    
    // 获取下一个排列
    bool hasNext = next_permutation(a, a + 3);
    cout << "第一次next后: " << a[0] << " " << a[1] << " " << a[2] << " (返回值: " << hasNext << ")" << endl;
    
    hasNext = next_permutation(a, a + 3);
    cout << "第二次next后: " << a[0] << " " << a[1] << " " << a[2] << " (返回值: " << hasNext << ")" << endl;
    
    return 0;
}
```

**输出结果**：
```text
初始: 1 2 3
第一次next后: 1 3 2 (返回值: 1)
第二次next后: 2 1 3 (返回值: 1)
```

<br />

## 4.两个重要规则

### 规则一：必须配合 ``do...while`` 遍历所有排列

如果你像下面这样写普通的 ``while`` 循环：
```cpp
while(next_permutation(a, a + 3)) {
    // 打印 a
}
```
你会发现，它**直接跳过了最初的 ``1 2 3``**，打印的第一项是 ``1 3 2``。因为 ``next_permutation`` 一上来就把 ``1 2 3`` 改成了 ``1 3 2``，然后才判断条件进入循环。

为了不漏掉初始状态，标准写法是使用 ``do...while``：
```cpp
do {
    // 处理当前排列 a
} while(next_permutation(a, a + 3));
```
这样就能保证先处理当前的排列，再去生成下一个。

<br />

### 规则二：初始序列必须是有序的（升序）

``next_permutation`` 只能向“字典序增大”的方向寻找。如果你的初始数组是 ``2, 1, 3``，它只会找到 ``2 3 1``、``3 1 2``、``3 2 1``，然后就结束了，前面的 ``1 2 3`` 和 ``1 3 2`` 会被漏掉。

所以，在调用 ``next_permutation`` 之前，**务必保证数组是升序的**。如果不放心，可以先调用一次 ``sort(a, a + 3);``。

<br />

### 5.底层原理

``next_permutation``的核心目标是：**在所有比当前序列大的排列中，找出一个“刚好大一点”的序列（即最小的更大排列）**。

为了实现这个目标，它通过以下 **4 个步骤** 完成（假设数组长度为 ``N``）：

**第 1 步：从后往前找“拐点”**

从数组的末尾向前遍历，找到第一个满足 ``左边 < 右边`` 的相邻元素对。

记录下这个“左边”元素的位置为 ``i``。

*（潜台词：这意味着从 ``i+1`` 到数组末尾的这一段，已经是严格的降序了。）*

<br />

**第 2 步：从后往前找“替换者”**

再次从数组的末尾向前遍历，找到第一个比 ``arr[i]`` 大的元素。

记录下它的位置为 ``j``。

*（潜台词：因为后缀是降序的，从后往前找第一个大于 ``arr[i]`` 的数，刚好就是后缀中大于 ``arr[i]`` 的最小的数。）*

<br />

**第 3 步：交换**

将位置 ``i`` 和位置 ``j`` 上的两个元素互换。

<br />

**第 4 步：后缀反转**

将位置 ``i+1`` 到数组末尾的这段序列反转（降序变升序）。

*（潜台词：因为前面找到了最小的替换者，所以后面的部分升序排列，就能保证这是“最小的更大排列”。）*

<br />

#### 具体例子演示

假设当前的排列是：``1 4 3 2``

**1. 找拐点**：

从后往前看：

- ``3 < 2`` 吗？否。
- ``4 < 3`` 吗？否。
- ``1 < 4`` 吗？是！

所以 ``i`` 指向 ``1``（索引为``0``）。此时 ``1`` 后面的序列 ``4 3 2`` 是降序的。

<br />

**2. 找替换者**：

从后往前找第一个大于 ``1`` 的数：

- ``2 > 1``，是！

所以 ``j`` 指向 ``2``（索引为``3``）。

<br />

**3. 交换**：

把 ``1`` 和 ``2`` 交换，序列变成：``2 4 3 1``

<br />

**4. 反转后缀**：

把 ``i`` 后面的序列（即现在的 ``4 3 1``）反转，变成 ``1 3 4``。

最终得到的序列就是：``2 1 3 4``

你可以对比一下 ``1 4 3 2`` 和 ``2 1 3 4``，后者确实是大于前者且最接近它的排列。

<br />

#### 特殊情况（已经是最大排列）

如果当前的序列已经是最大的了，比如 ``4 3 2 1``：

- 在第 1 步“找拐点”时，从头到尾都没找到 ``左边 < 右边`` 的情况。
- 此时函数知道当前已经是最大排列，它会直接返回 ``false``。
- 同时，它会顺手把整个数组反转（变成最小的升序 ``1 2 3 4``），方便你从头开始新一轮的循环。

<br />

#### 时间复杂度

通过这个算法可以看出，它最多只需要遍历两次数组，再进行一次反转，没有嵌套循环，因此时间复杂度是优秀的 O(N)。
