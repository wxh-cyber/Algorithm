# ``sort``的底层实现

<hr />

C++ 标准库中的 ``std::sort`` 底层并不是采用单一的排序算法，而是采用了一种**混合排序算法**，在大多数标准库实现（如 GCC 的 libstdc++、LLVM 的 libc++、MSVC 的 STL）中，这种算法被称为 **Introsort（内省排序）**。

Introsort 的设计哲学是“**取长补短**”，它结合了三种经典排序算法的优点：**快速排序、堆排序**和**插入排序**。

## 1. 三大核心组件
- **快速排序**：
    - **优点**： 绝大多数情况下极快，常数因子小，原地排序不需要额外空间。
    - **缺点**： 最坏时间复杂度会退化到 O(N²)（例如数组已经有序，或者每次选的 pivot 都极度不平衡），且递归深度可能过深导致栈溢出。

- **堆排序**：
    - **优点**： 最坏时间复杂度严格保证为 O(NlogN)，不需要额外空间。
    - **缺点**： 常数因子较大，实际运行速度比快排慢很多（因为缓存不友好，频繁跳跃访问内存）。

- **插入排序**：
    - **优点**： 在数据量很小，或者数据**基本有序**时，效率极高，常数极小。
    - **缺点**： 数据量大时时间复杂度为 O(N²)，不可接受。

<br />

## 2. Introsort 的协同工作流程

``std::sort`` 在执行时，会像带有一个“仪表盘”一样，根据当前数据的状态在这三种算法间切换：

**第一阶段：快速排序主导**

一开始，``std::sort`` 使用快速排序进行划分。为了防止最坏情况，现代 C++ 标准库通常采用**三点中值法**（取首、尾、中三个元素的中间值）来选取 pivot（基准值），这极大降低了退化到 
O(N²) 的概率。

<br />

**第二阶段：内省机制触发 -> 切换为堆排序**

这是 Introsort 得名的原因（Introspective 是内省/自省的意思）。在快排递归的过程中，算法会记录当前的**递归深度**。

- 如果发现递归深度超过了某个阈值（通常是 2×log2(N)），算法就会“内省”到：当前的快排可能陷入了最坏情况（分区极度不平衡）。
- 此时，算法会在当前未排好的子数组上**放弃快排，切换为堆排序**。
- **效果**： 强行将最坏时间复杂度拉回到 O(NlogN)，满足了 C++ 标准对 ``std::sort`` 复杂度的强制要求。

<br />

**第三阶段：小区间优化 -> 切换为插入排序**

在快排不断划分区间的过程中，当子数组的长度变得非常小（通常阈值设定为 **16**）时，继续递归快排的开销（函数调用、栈空间）已经超过了排序本身的开销。

- 此时，算法不再继续递归，而是直接返回。
- 注意： 此时整个数组并不是完全有序的，而是处于“**宏观上有序，微观上局部无序**”的状态。
- 当最外层的递归全部返回后，``std::sort`` 最后会对整个数组执行一次插入排序。因为此时数组已经基本有序，插入排序的效率接近 O(N)，常数极小，速度飞快。
