### getline
**语法构成**：
```cpp
getline(cin, str);
```
**功能**：
从输入流中读取一行字符串，直到遇到换行符为止。

**特点**：
1. 可以读取包含空格的字符串。
2. 不会忽略行首的空格。
3. 可以读取空行。

**底层**：
配合 ``std::string`` 使用，自动动态扩容，绝对安全，不存在缓冲区溢出。

**⚠️ 竞赛最大踩坑点**：如果你在前面用了 ``cin >> n`` 或 ``cin >> s``，输入缓冲区里会残留一个 ``\n``。直接调用 ``getline`` 会读到这个空行！

**解决方案**：在 ``getline`` 前加一句 ``cin.ignore()``。
```cpp
int n;
string line;
cin>>n;
cin.ignore();
getline(cin,line);
```
更稳一点儿可以写：
```cpp
cin.ignore(numeric_limits<streamsize>::max(),'\n');
```
``numeric_limits<streamsize>::max()`` 表示一直忽略到换行为止。

需要头文件：
```cpp
#include <limits>
```

**常用模板**：
```cpp
#include<iostream>
#include<string>
using namespace std;

int main(){
    int n;
    string s;

    cin>>n;
    cin.ignore();

    getline(cin,s);

    cout<<n<<endl;
    cout<<s<<endl;
    return 0;
}
```

### cin.getline
**语法构成**：
```cpp
cin.getline(char_array,max_size,stop_char);
```
- ``char_array``：一个字符数组（``char buf[105]``），用来存放读进来的数据。
- ``max_size``：最大读取字符数（包含结尾的 ``\0``）。
- ``stop_char``：指定一个字符，读到这个字符就停止（比如逗号 ,）。这个字符会被从流中提取并丢弃，不会被放进数组里。

**作用**：
读取一整行存入字符数组，遇到换行符停止（同样不读入 \n）。

**区别**：
必须指定最大长度 ``size``，如果一行超过 ``size-1``，会截断并触发错误状态（后面再读就全废了）。

**适用场景**：
题目强制要求用 C 风格字符数组 ``char s[]``，或者需要配合 ``sscanf`` 进行复杂格式化提取时。

**用法演示**：
```cpp
#include<iostream>
using namespace std;

int main(){
    char buf[105];
    
    //输入：Hello Algorithm contest
    cin.getline(buf,105);
    cout<<buf<<endl;       //输出：Hello Algorithm contest

    //输入：2026,10,01
    cin.getline(buf,105,',');
    cout<<buf<<endl;       //输出：2026（逗号不输出）

    return 0;
}
```
#### ⚠️ 算法竞赛的致命踩坑点（重点！）
在 OJ（在线评测）中，因为用错 ``cin.getline`` 导致的 WA（答案错误）或 RE（运行错误）简直不要太多。主要有三大坑：

**坑 1：前置换行符污染（和 ``getline`` 犯一样的错）**
如果你在 ``cin.getline`` 之前用了 ``cin >>`` 读数字或单词，缓冲区里会留下一个 ``\n``。``cin.getline`` 看到 ``\n`` 会直接认为这一行是空的，立刻结束读取。
```cpp
int n;
char s[105];
cin>>n;
//cin.getline(s,105);   // 错误！s 会读到一个空字符串 ""
cin.ignore();           // 正确！必须加这一句吃掉换行符
cin.getline(s, 105);    // 正确读取
```

**坑 2：对 ``max_size`` 的理解错误（差一错误 Off-by-one）**
如果你定义了 ``char buf[100];``，那么 ``max_size`` **必须写 100**，不能写 99 或 101。

- 如果写 ``99``：它最多读取 99 个字符，最后自动补 ``\0``（刚好占满 100 个位置）。
- 如果你输入了 105 个字符，它**不会报错或溢出**，而是直接截断，只保留前 99 个字符加上 ``\0``，剩下的字符继续留在输入流里。

**坑 3：截断导致的“死亡状态”—— 最阴险的坑！**
假设你的数组开了 ``char buf[10];``，但你输入了一行 50 个字符的字符串。

``cin.getline(buf, 10)`` 会读入前 9 个字符，补上 ``\0``，然后停止读取。

**关键来了**：此时 ``cin`` 对象内部会被设置一个**错误标志位**。一旦这个标志位被设置，**后续所有的 ``cin`` 输入（无论是 ``cin >>`` 还是 ``cin.getline``）都会永久失效，直接跳过！**

**错误复现：**
```cpp
char s1[10],s2[105];
// 输入第一行: 1234567890ABCDEF... (超过10个字符)
cin.getline(s1, 10); // 读入 "123456789"，流进入错误状态

// 输入第二行: Hello World
cin.getline(s2, 105); // 什么都不读！s2 是空的！程序直接废了！
```

**解决方案（如果非要截断）：**
```cpp
cin.getline(s1,10);
if(cin.fail()){                //检查是否发生了截断
    cin.clear();               //1.清除错误标志位，让cin恢复工作
    cin.ignore(1000,'\n');     //2.把这一行剩下没读完的垃圾字符全部扔掉
}
```
>注：在算法竞赛中，没人愿意写这么繁琐的代码。因此，我们一般不使用 ``cin.getline``。

### cin.get()
**作用**：每次只读取**单个字符**（包括空格和换行符），将其作为``int``类型返回。
**为什么返回``int``而不是``char``？**
因为当读取到文件结尾（EOF）时，它会返回一个特殊的负数（通常是``-1``），``char``存不下这个值，会导致死循环。
**适用场景**：当你需要手动逐个字符解析输入，或者只为了吃掉一个换行符时（替代 ``cin.ignore()``）。
```cpp
char c=cin.get();           //读取任意一个字符
```

#### 用法一、读取单个字符（重点掌握）
**1. 无参数形式：int c=cin.get()**
```cpp
int c;
while((c=cin.get())!=EOF){           //经典的逐字符读取模板
    if(c==' ') break;
    cout<<(char)c;
}
```

**2. 带一个参数形式：cin.get(char c)**
**作用**：读取一个字符存入变量 ``c`` 中。返回值是 ``cin`` 对象本身（一般没人用它的返回值）。
**用法示例**：
```cpp
char c;
cin.get(c);                //等价于C语言的c=getchar()
```

**🏆 竞赛实战场景：什么时候用它读单字符？**
1. **精准吞噬换行符**：代替 ``cin.ignore()``。当你用 ``cin >> n`` 读完后，直接写 ``cin.get()``; 把换行符吃掉，代码短且快。
2. **读取无空格的字符矩阵（网格图）**：
```cpp
//读入一个5×5的字母迷宫
char maze[5][5];
for(int i=0;i<5;i++){
    for(int j=0;j<5;j++){
        maze[i][j]=cin.get();             //逐个读取字母
    }
    cin.get();                            //吃掉每行最后的换行符
}
```

3. **手写快读底层**：很多快读模板底层就是用 ``cin.get()`` 或 ``getchar()`` 来跳过前导空格并读取数字的。

#### 用法二：读取字符串到字符数组（避坑指南）
``cin.get`` 还可以重载为读取一整行，这也就是大家最容易把它和 ``cin.getline`` 搞混的地方。
**语法：**
```cpp
cin.get(char_array,max_size);
```
**参数**：和 ``cin.getline`` 完全一样。目标数组、最大长度（包含 ``\0``）、结束符（默认是 ``\n``）。


**💣 最核心的区别（面试/坑点常客）：**
``cin.get`` 读取到结束符（如 ``\n``）时，会停止读取，但**【不会把结束符从缓冲区里拿走】**！
而 ``cin.getline`` 读取到 ``\n`` 时，会把它拿走并扔掉。
**错误复现（连续读两行的血案）：**
```cpp
char s1[105],s2[105];

//输入第一行：Hello
cin.get(s1,105);
//此时s1里面是"Hello"，但输入缓冲区里还留着'\n'！

//输入第二行：World
cin.get(s2,105)
//灾难发生！cin.get 一上来就看到了缓冲区里的 "\n"，直接认为 s2 读取结束！
// s2 变成了一个空字符串 ""，而 "World" 还留在缓冲区里没被读。
```
**如果非要用 cin.get(buf) 读多行，必须手动清理：**
```cpp
cin.get(s1,105);
cin.get();
cin.get(s2,105);
```

### 终极对比表
| 函数 | 读单字符 | 读一行到数组 | 遇到\n怎么处理 | 竞赛推荐度 |
| :--- | :--- | :--- | :--- | :--- |
| ``cin.get()`` | ✅ (返回``int``) | ❌ | - | ⭐⭐⭐⭐⭐ (极常用) |
| ``cin.get(c)`` | ✅ (存入``char``) | ❌ | - | ⭐⭐⭐⭐ |
| ``cin.get(buf)`` | ❌ | ✅ | 留在缓冲区里不吃掉 | ❌ (极不推荐，太容易出错) |
| ``cin.getline()`` | ❌ | ✅ | 吃掉并扔掉 | ⭐⭐ (能用，但不如 ``getline(cin, string)``) |
| ``getchar()`` | ✅ (C语言) | ❌ | - | ⭐⭐⭐⭐⭐ (最快) |

### gets(char_array) —— ❌ 绝对禁止使用
**作用**：读取一整行到字符数组，遇到 ``\n`` 停止。
**为什么不用：极其危险！** 它不检查数组边界，如果输入长度超过数组大小，直接缓冲区溢出。在 C++11 中被彻底废弃，现在的主流评测机（如 Codeforces, 洛谷）基本都会报错或直接 CE（编译错误）。

### fgets(char_array, size, stdin)
**作用**：``gets`` 的安全替代版。最多读 ``size-1`` 个字符。
**⚠️ 竞赛最大踩坑点**：与 ``getline`` 不同，``fgets`` 会把结尾的 ``\n`` 一起读进数组里！ 如果不去掉，做字符串比较（如 ``strcmp``）或哈希时会出错。
**解决方案**：手动把最后一个字符改成 ``\0``。
```cpp
char buf[105];
fgets(buf,105,)
```
