## 输入输出流

<hr />

一般规则很简单：<br />

- `ios::sync_with_stdio(false); cin.tie(0);` 主要用于 加速 `cin/cout`
- 如果你只用 `cin/cout`，通常可以 关闭
- 如果你要混用 `scanf/printf` 和 `cin/cout`，通常应该 开启，或者干脆别混用

<hr />

### 什么时候关闭？

适合这些情况：<br />

- 题目数据量大，担心 `cin/cout` 太慢
- 全程只用 C++ 风格输入输出：`cin`、`cout`
- 竞赛/刷题里需要更快的 IO 操作

常见写法：

```cpp
ios::sync_with_stdio(false);
cin.tie(0);
```

意思是：<br />

- `ios::sync_with_stdio(false);`：取消 C 和 C++ 标准流同步，提高速度
- `cin.tie(0);`：解除 cin 和 cout 的绑定，避免每次输入前都自动刷新输出

<hr />

### 什么时候开启？

适合这些情况：<br />

- 你要混用 `printf/scanf` 和 `cout/cin`
- 你更在意输出顺序稳定，而不是极限性能
- 写交互题但你不太熟悉刷新机制时，要更谨慎

默认其实就是“开启同步”的，所以如果要开启，通常什么都不用写；或者写：

```cpp
ios::sync_with_stdio(true);
```

<br />

**重点结论:**

1. 只用 `cin/cout`：建议关闭
2. 只用 `scanf/printf`：不用管它
3. 混用两套 IO：建议开启同步，或者不要混用
4. 交互题：可以关闭，但要手动刷新，比如 `cout << x << endl;` 或 `cout << x << flush;`

<br />

### 为什么混用时不要关闭？

因为关闭同步后，`cout` 和 `printf` 各自有自己的缓冲区，输出先后顺序可能和你写代码的顺序不一致。

例如：

```cpp
ios::sync_with_stdio(false);
cout << "A";
printf("B");
```

你可能以为输出是：

```cpp
AB
```

但实际顺序可能乱掉。
