# ``size_t``

<hr />

在 C++ 中，``size_t`` 是一种非常基础且重要的**无符号整型**数据类型。

简单来说，它的核心用途是：**用来表示对象的大小、数组的索引或字节数**。

<hr />

## 1.``size_t``是什么？

- **定义位置**：定义在 C++ 标准头文件 ``<cstddef>`` 中（以及其他头文件如 ``<cstdio>``, ``<cstdlib>``, ``<cstring>``, ``<string>``, ``<vector>`` 等）。
- **本质**：它是平台相关的别名。
    - 在 **32 位系统**上，通常是 ``unsigned int``（32位，最大值约 42 亿）。
    - 在 **64 位系统**上，通常是 ``unsigned long long``（64位，最大值极大）。
- **关键特性**：它是无符号的，意味着它永远是非负数（≥0）。

<hr />

## 2.主要用途

``size_t`` 的设计初衷是为了让代码具有更好的**可移植性**和**安全性**。它在以下场景中无处不在：

### A.``sizeof``运算符的返回值

这是最常见的用途。``sizeof`` 操作符返回的类型就是 ``size_t``。
```cpp
int arr[10];
size_t size = sizeof(arr); // 正确
// int size = sizeof(arr);  // 可能会有警告，因为类型不匹配
```

<br />

### B.STL容器的``.size()``方法

标准模板库（STL）中的容器（如 ``std::vector``, ``std::string``, ``std::map``）的 ``size()`` 方法返回的都是 ``size_t``。

```cpp
#include <vector>
#include <string>

std::vector<int> v = {1, 2, 3};
std::string s = "hello";

size_t v_size = v.size(); // 3
size_t s_len = s.length(); // 5
```

<br />

### C.容器与数组的索引

因为 ``size_t`` 是无符号的且足够大，它非常适合作为数组的下标。
```cpp
for (size_t i = 0; i < v.size(); ++i) {
    // 访问 v[i]
}
```

<br />

### D.内存分配函数

动态内存分配函数（如 ``malloc``, ``new``, ``operator new``）接受 ``size_t`` 作为参数来指定要分配的字节数。
```cpp
void* ptr = malloc(100 * sizeof(int)); // 这里的 100 实际上会被隐式转换为 size_t
```

<hr />

## 3.为什么要用 ``size_t`` 而不是 ``int``？

如果你使用 ``int`` 来处理大小或索引，可能会遇到以下两个严重问题：

**原因一：容量限制（可移植性）**

- 在 64 位系统上，程序可能使用超过 4GB 的内存。
- 32 位 ``int`` 最大只能表示到约 2×10^9（20亿），如果数组大小超过 20 亿，``int`` 就会溢出变成负数。
- ``size_t`` 在 64 位系统上是 64 位的，可以轻松处理巨大的内存空间。

**原因二：语义正确性（负数问题）**

- 数组的下标、字符串的长度、内存的大小，在物理上**绝不可能是负数**。
- 使用 ``size_t`` 可以在类型层面表达“这是一个非负数”的语义，防止意外的负值赋值（尽管编译器可能只给警告）。

<hr />

## 4.常见的陷阱与注意事项

虽然 ``size_t`` 很好用，但因为它是一个**无符号类型**，初学者很容易掉进坑里：

<br />

### 陷阱一：有符号与无符号的比较

这是 C++ 中最经典的警告来源之一。当你把 ``int``（有符号）和 ``size_t``（无符号）进行比较时，``int``会被隐式转换为 ``size_t``。
```cpp
std::vector<int> v = {1, 2, 3};

// 错误示范：int i
for (int i = 0; i < v.size(); ++i) { 
    // 编译器警告：有符号/无符号比较
    // 如果 v 为空，v.size() 是 0。如果 i 计算变成 -1，比较时会变成巨大的正数，
    // 导致死循环或逻辑错误。
}
```

**解决方法**：循环变量直接使用 ``size_t``，或者使用 C++11 的 ``auto``。
```cpp
// 推荐
for (size_t i = 0; i < v.size(); ++i) { ... }

// 或者使用 auto（C++11）
for (auto i = 0u; i < v.size(); ++i) { ... } // 0u 表示 unsigned int
```

<br />

### 陷阱二：反向循环（下溢）

由于 ``size_t`` 永远不能为负数，反向循环时不能写成 ``i >= 0``，因为那永远为真，会导致死循环（当 ``i``为 0 时减 1，它会变成最大的正数，即回绕）。

```cpp
// 错误：死循环！
for (size_t i = v.size() - 1; i >= 0; --i) { 
    // 当 i == 0 时，执行 --i，i 变成 18446744073709551615 (MAX)
}

// 正确做法 1：使用 while 循环
size_t i = v.size();
while (i-- > 0) {
    // 使用 i
}

// 正确做法 2：使用有符号类型（前提是你确定不会越界）
for (int i = static_cast<int>(v.size()) - 1; i >= 0; --i) { ... }
```

<hr />

## 总结

**``size_t`` 是 C++中表示“大小”的标准语言**。

- **做加法/正向遍历**：首选 ``size_t``。
- **做减法/反向遍历**：要极其小心下溢，通常改用 ``int`` 或特殊的循环写法。
- **最佳实践**：当你的变量代表“长度”、“个数”、“字节大小”时，请毫不犹豫地使用 ``size_t``。
